Aroma's decomposition

Development of an Automated Molecular Analysis Tool for Aroma Composition

A leading industrial company aims to develop a tool that enables the study of the molecular composition of aromas in the market. The primary objectives are to stay competitive with other producers and implement quality control measures throughout their supply chain. By optimizing their processes, the company seeks to reduce manual intervention time in these analytical tasks.

To address this challenge, we developed a fully automated end-to-end pipeline for data ingestion and preprocessing. The data, originating from a Gas Chromatography-Mass Spectrometry (GC-MS) instrument, consists of three-dimensional tabular datasets. A key component of our approach is the implementation of sparse coding techniques to detect molecules present in aromas. This method enables the identification of compounds by leveraging standardized molecular spectra, enhancing the accuracy and efficiency of aroma analysis.

Overview

Sparse coding provides a powerful, automated approach to molecular identification in GC-MS analysis by leveraging the NIST spectral database. It improves detection accuracy, reduces manual workload, and enhances the overall efficiency of chemical analysis in industrial and research applications. For more information on sparse coding techniques, visit the SPORCO GitHub repository.

Activities:

  • Preprocessed raw chromatographic signals to improve data quality and ensure consistency.
  • Applied sparse coding techniques to encode chromatogram peaks into low-dimensional representations.
  • Identified key molecular markers responsible for differentiating aroma profiles.

How it works:

Data Preprocessing

  • The raw GC-MS signal (chromatograms and mass spectra) is collected.
  • Baseline correction, noise filtering, and peak detection are performed to extract relevant spectral information.
Dictionary Construction Using NIST Database
  • The NIST database contains standardized mass spectra of known molecules.
  • These spectra are used to construct a dictionary where each basis function (column) represents a reference spectrum of a molecule.
Sparse Decomposition of the Sample Spectrum
  • Given an unknown GC-MS spectrum, sparse coding finds a sparse combination of dictionary entries that best reconstructs the observed signal.
  • This is typically formulated as:
X = D ⋅ A + ε

In this equation:

  • X represents the observed spectrum, which is the input GC-MS data.
  • D stands for the dictionary, derived from NIST spectra.
  • A is the sparse coefficient matrix, showing the contribution of each molecule.

L1-norm minimization is used to ensure only a few dictionary elements (molecules) contribute to the reconstruction.

Molecule Identification

  • The nonzero entries in A indicate the presence of corresponding molecules from the NIST database.
  • The identified molecules are validated using retention times and intensity patterns.
